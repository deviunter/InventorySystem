## RUSSIAN README

# СИСТЕМА ИНВЕНТАРЯ ABYSSWHISPER
Система инвентаря ABYSSWHISPER - это духовный наследник системы инвентаря After Darkness, её улучшенная и отрефакторенная версия, вдобавок перенесённая полностью на C++
Конкретно этот репозиторий расчитан в качестве временного портфолио, а система на данном этапе разработки хоть и полностью модульная, в первую очередь заточена строго под ABYSSWHISPER.
Однако, в последствии, если в этом будет необходимость я планирую внести форк этой системы в REFLECTION ENGINE.
Кратко о том, что послужило референсом, для этого инвентаря. Главный референс это инвентарь игры Alan Wake 2. Все подобноро рода инвентари по сути одинаковы, но то, как именно (на мой взгляд) в AW2 инвентарь взаимодействовал
с другими системами и то, как это работало на атмосферу там, мне было референсом.

# ПЛАНЫ НА БУДУЩЕЕ
1. Полностью доделать систему под ABYSSWHISPER

  1A. Доделать дочерний класс, предназначенный для ящиков с лутом, которые не "выбрасывают" лут, спавня предметы после открытия, а открытия инвентаря этого объекта и получение предметов через виджет, как это реализовано в, S.T.A.L.K.E.R например.
  
  1B. Доделать дочерний класс, предназначенный для хранилища - особого типа инвентаря в который нужен только для сбрасываения лишних предметов, поскольку сам инвентарь не самостоятелен, этот тип компонента будет крепиться либо к GameMode либо к особой кастомной системе AftermathSystem.
  
  1C. Доделать сами предметы - дочерние классы UItemBase.
  
  1D. Доделать UI.
  
  1E. Доделать систему ресурсов, оберегов, ключевых предметов и QuickAccess ячейки. - короче все подсистемы инвентаря UPlayerInventoryComponent.
  
  1F. Доделать подбор предметов.
  
2. Доработать базовый функционал UInventoryComponent и других основных классов Inventory System и подготовить её для адаптации и интеграции её в Reflection Engine.

# O REFLECTION ENGINE

В тексте несколько раз упоминалось о Reflection Engine и в этой небольшой справке описано, что он из себя представляет.
Reflection Engine — это модульный фреймворк для Unreal Engine 5, разработанный как консолидированная кодовая база готовых к производству игровых систем. Он служит нескольким целям:
Внутренняя стандартизация: Обеспечивает единую основу для всех проектов Skydream Interactive, гарантируя согласованность и снижая затраты на разработку.
Техническое портфолио: Выступает в качестве наглядного представления основных инженерных и архитектурных компетенций в проектировании игровых систем.
Коммерческий потенциал: Предлагает базовый уровень для потенциального будущего коммерческого продукта, ориентированного на быстрое прототипирование и разработку сюжетных и приключенческих жанров.
Фреймворк построен с акцентом на модульность, проектирование на основе данных и чистую интеграцию C++/Blueprint, что позволяет масштабировать и поддерживать разработку проектов.
